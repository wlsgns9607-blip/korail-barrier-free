# 코레일톡 배리어프리 온·오프라인 통합 서비스 기획

40대 이상·고령층 승객의 현장 VOC를 바탕으로 작성한 코레일톡 서비스 기획 포트폴리오입니다.
모바일 앱 UX 개선과 승강장·열차 내부 공간 서비스 기획을 하나로 잇는 시연용 원페이지 웹사이트입니다.

## 이 기획의 차별점

일반적인 "고령층 접근성 개선" 제안은 대부분 글자 크게 보기, 고대비 모드처럼
**화면 안에서 끝나는 UI 조정**에 머무릅니다. 이 기획은 세 가지 지점에서 다르게 접근합니다.

1. **화면 개선이 아니라 여정 전체 재설계**
   앱 UX, 승강장 모니터, 열차 내부 태블릿을 하나의 문제(승객이 자리를 못 찾고,
   물어볼 사람이 없다)를 푸는 하나의 시스템으로 묶었습니다. 서비스 기획자가
   디자이너의 영역(화면)과 운영자의 영역(현장 인프라)을 동시에 설계했다는 점이
   일반적인 UX 개선안과 가장 크게 갈리는 지점입니다.

2. **사후 대응이 아니라 사전 예방 설계**
   승무원 태블릿은 "이미 발생한 불편"에 답하지만, 플랫폼 모니터 매핑은
   "불편이 발생하기 전"에 동선을 알려줍니다. 문제 발생 이후 대응 창구를 늘리는
   방식이 아니라, 문제 자체가 생기지 않도록 탑승 전 단계를 설계에 포함했습니다.

3. **감성적 배려가 아니라 정량적 비즈니스 논리로 제시**
   "어르신을 위한 배려"라는 정성적 명분에 머물지 않고, 인건비 절감·정시성 확보·
   ESG 사회(S) 지표 상승이라는 코레일의 실제 경영 지표와 연결했습니다.
   Before/After 비교와 전광판 스타일 카운트업 수치는 이 논리를 화면에서도
   시각적으로 증명하기 위한 장치입니다.

4. **말로 설명하는 기획서가 아니라 눌러보는 기획서**
   PPT 캡처나 손그림 목업 대신, 실제로 동작하는 인터랙티브 데모(실버 모드 확대,
   좌석 하이라이트 + 실제 음성 재생, AI 챗 시뮬레이션)를 구현해 면접관이 직접
   조작하며 설득되도록 만들었습니다. 이 자체가 "구현까지 고려하는 기획자"라는
   포지셔닝의 증거이기도 합니다.

## 파일 구성

```
├── index.html      메인 페이지 (구조/콘텐츠)
├── styles.css       디자인 (레이아웃, 컬러, 애니메이션)
├── script.js         인터랙션 로직 (데모 3종, 카운트업 애니메이션)
└── README.md        본 문서
```

세 파일은 반드시 같은 폴더 안에 함께 두어야 합니다. `index.html`이 상대 경로로
`styles.css`와 `script.js`를 불러오기 때문입니다.

## 실행 방법

별도 설치나 서버 없이 바로 열 수 있습니다.

1. 세 파일을 같은 폴더에 저장합니다.
2. `index.html`을 더블클릭하거나, 브라우저(Chrome 권장)로 열면 됩니다.
3. 인터넷 연결이 필요합니다 — 폰트(Pretendard, JetBrains Mono)를 CDN에서 불러오고,
   좌석 안내 음성은 브라우저 내장 음성 합성 기능(Web Speech API)을 사용합니다.
   연결이 없어도 페이지는 열리지만 기본 시스템 폰트로 표시되고 음성 안내는 재생되지 않습니다.

면접이나 발표 현장에서는 노트북/태블릿에 파일을 미리 옮겨두고,
가능하면 발표 전 Wi-Fi 연결 상태를 한 번 확인해두는 것을 권장합니다.

## 페이지 구성

1. **히어로** — 기획 배경과 현장 페인 포인트 4가지 요약
2. **Before / After** — 현재 앱 vs 개선안 비교, 정량 지표(터치 성공률 등) 애니메이션
3. **핵심 기능 3종** — 들리는 배리어프리 모드 / 플랫폼 모니터 매핑 / 차칸 AI 승무원 태블릿
4. **비즈니스 효과** — 인건비 절감·정시성 확보·ESG 점수 상승을 전광판 스타일 숫자로 시각화
5. **인터랙티브 데모** — 아래 3개 탭 실제 동작

   | 탭 | 내용 |
   |---|---|
   | 실버 모드 토글 | 버튼이 실시간으로 커지는 효과 시연 |
   | 내 좌석 확인 | 좌석 배치도 위 내 자리 하이라이트 + 실제 음성 안내 재생 |
   | 차칸 AI 태블릿 | 질문 클릭 → AI 답변 채팅 시뮬레이션, 복잡 문의는 승무원 화상 연결로 전환 |

## 커스터마이징 가이드

- **문구 수정**: `index.html`에서 해당 섹션의 한글 텍스트를 직접 수정하면 됩니다.
- **색상 변경**: `styles.css` 최상단 `:root { --navy: ...; --amber: ...; }` 부분의
  변수 값만 바꾸면 전체 페이지 색상이 함께 바뀝니다.
- **지표 숫자 변경**: `index.html`의 `data-target` 속성 값(예: `data-target="35"`)을
  원하는 수치로 바꾸면 카운트업 애니메이션에 자동 반영됩니다.
- **데모 응답 문구 변경**: `script.js`의 `responses` 객체 안 텍스트를 수정하면
  AI 태블릿 데모의 질문/답변 내용을 바꿀 수 있습니다.

## 참고 사항

- 비즈니스 효과 수치(터치 성공률, 인건비 절감률 등)는 실측 데이터가 아닌
  현장 VOC와 기획 가정에 근거한 추정치입니다. 실제 제안 시에는 검증이 필요합니다.
- 모든 화면(스마트폰 목업, 좌석 배치도, 태블릿 UI)은 이미지 파일 없이
  CSS와 SVG만으로 구현되어 있어 파일 용량이 가볍고 어떤 환경에서도 동일하게 보입니다.
